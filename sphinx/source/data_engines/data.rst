Будет сервер для POST запросов с инициализацией протокола . Возможно так:

- при установке связи с сервером клиент будет передавать POST запрос с файлом протокола,
    можно json, yaml,  или Python dictionary. dict готов уже,
    но удобнее будет с json,
    я добавлю в парсер протокола метод, возвращающий json

- сервер имеет свой вариант протокола, он сверяет его с переданным протоколом,
    отвечает ок и ждет POST с данными в json или ошибка, тогда ждет протокол снова

- получив POST  с данными, возвращает json с прогнозом
    или статусом ошибки

- сообщения об ошибках
    сделаем по ходу.

- Можно статус ошибки передавать в протоколе клиента при инициализации,
    сервер не будет его парсить, просто вернет ту ошибку,
    которую хочет видеть клиент,
    сервер также  может вести свои собственные логи ошибок (в REDIS, к примеру.)

- Для модели нужны только типы данных,
    остальное в протоколе для тестов
